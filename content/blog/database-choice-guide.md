# Какую базу данных выбрать

Один из первых архитектурных вопросов — какая база данных нужна. Ответ: «зависит от задачи». Давайте разберёмся.

## Типы БД и когда они нужны

### Реляционные (SQL)

- **PostgreSQL** — универсальный выбор. Сложные запросы, транзакции, надёжность.
- **MySQL** — похож на PostgreSQL, но попроще. Хорошо для web-приложений.
- Когда нужны: сложные связи между данными, транзакции, строгая схема.

### Документные (NoSQL)

- **MongoDB** — документы без жёсткой схемы. Гибко, но требует дисциплины.
- Когда нужны: разные типы данных, быстрая разработка, неопределённые требования.

### Key-Value

- **Redis** — сверхбыстрый. Кэш, сессии, очереди.
- Когда нужны: кэширование, подсчёт метрик, очереди.

### Поисковые

- **Elasticsearch** — полнотекстовый поиск, аналитика логов.
- Когда нужен: поиск по тексту, дашборды.

### Колоночные

- **ClickHouse** — аналитика на больших объёмах.
- Когда нужна: аналитика, метрики, Big Data.

## Простой чеклист выбора

- Нужны транзакции? → PostgreSQL/MySQL
- Неизвестна структура данных? → MongoDB
- Нужен кэш? → Redis
- Нужен поиск? → Elasticsearch
- Нужна аналитика? → ClickHouse
- Всё сразу? → Несколько БД вместе.

## Миграция между БД

Смена БД — это больно. Лучше сразу думать наперед. Но если пришлось — используйте инструменты миграции, тестируйте на копии, планируйте downtime.
