## Blockchain — инструмент, не цель

В 2024–2026 рынок зрелее: меньше проектов «ради токена», больше **аудит трейсов, расчёты между компаниями, цифровые права на документы**. Но по-прежнему 70% RFP можно закрыть централизованной БД быстрее и дешевле.

## Decision matrix

| Критерий | Нужен blockchain? | Альтернатива |
|----------|-------------------|--------------|
| Несколько независимых участников | Да (часто) | — |
| Один владелец данных | Нет | PostgreSQL |
| Неизменяемый audit trail | Иногда | WORM storage + подпись |
| Мгновенные micropayments в публичной сети | Да | L2 / private chain |
| 100k TPS внутренние операции | Нет | OLTP + queue |

## Кейсы с измеримым эффектом

**Supply chain:** фиксация событий (отгрузка, таможня, приёмка) — сокращение споров на 20–40% времени разбора (по пилотам в логистике).

**B2B settlements:** escrow smart contract между покупателем и поставщиком — меньше посредников, но нужна юридическая обвязка off-chain.

**Document notarization:** hash документа в chain, полный файл off-chain — доказательство неизменности без хранения ПДн в сети.

**DeFi / Web3 продукта:** см. [кейс DeFi-дашборда](/cases/fintech-portal) — кастоди, кошельки, compliance UI.

## Архитектура пилота (упрощённо)

```
[ERP/WMS] --> [Integration API] --> [Private ledger nodes]
                                        |
[Web dApp] <-- wallet connect ----------+--> [Indexer] --> Analytics
```

На пилоте — **2–3 узла**, один smart contract, ограниченный набор событий. Не переносить весь ERP в chain.

## ROI-шаблон

```
Годовая экономия = (часы на споры × ставка) + (штрафы избежанные) + (оборот быстрее)
Затраты = разработка + инфра + аудит контракта + поддержка узлов
Payback = затраты / (экономия/12)
```

Если payback > 18 месяцев без стратегического актива — пересмотреть scope.

## Риски

- Смарт-контракт без аудита — эксплойт = репутационный и финансовый удар.  
- Gas spikes в публичных сетях — UX ломается.  
- Key management — потеря ключей = потеря активов.  

## План пилота 10 недель

1. Юридический scope (1–2).  
2. Event model + contract (3–5).  
3. Integration API (6–7).  
4. Audit + testnet (8).  
5. Mainnet limited partners (9–10).
